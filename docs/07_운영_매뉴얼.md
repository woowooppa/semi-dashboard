```bash

```

### ① 값 재기 — 브라우저 탭 2개

**현물가** — `trendforce.com/price/dram/dram_spot`
→ **DDR4 8Gb (1Gx8) 3200** 평균값 (USD/chip)

**SOXX 후행 P/E** — `ishares.com` SOXX 페이지 → Portfolio Characteristics → **`P/E Ratio`**
→ ⚠ `Number of Holdings`(약 30)와 혼동 금지

### ② `data/manual.json`에 한 줄 추가

`entries` 배열 **맨 끝**에 붙인다. 앞 항목 끝에 **쉼표를 찍는 것**을 잊지 말 것.

```json
    {
      "date": "2026-07-27",
      "ddr4_8gb_spot_usd": 42.5,
      "soxx_pe_trailing": 65.2,
      "grade": "A",
      "src": "TrendForce / iShares",
      "note": "공급 타이트 지속, 서버 수요 중심"
    }
```

**규칙 넷**
1. `date`는 **조회한 날이 아니라 데이터 기준일**(as of)
2. 모르는 값은 아예 **쓰지 않는다** (`null`도 가능)
3. `src`는 반드시 남긴다 — 나중에 이 값을 의심할 때 유일한 단서
4. `note`는 숫자가 설명 못 하는 것만 — 그날의 시장 분위기

### ③ 저장 → 푸시

```bash
git add .
git commit -m "주간 갱신 2026-07-27"
git push
```

**푸시하면 Actions가 자동으로 재실행된다.** `manual.json` 변경이 트리거로 걸려 있다. 2분쯤 뒤 대시보드에 반영된다.

## 7.4 분기 1회 — 40분, 이게 본체다

주가나 현물가는 매일 봐도 판단이 안 바뀐다. **판단이 바뀌는 건 여기서다.**

### ① 3사 실적 정독

| 회사 | 소스 | 뽑을 것 |
|---|---|---|
| 삼성전자 | IR + 컨콜 (디일렉이 전문 게재) | DRAM/NAND Bit Growth, ASP, 가이던스 |
| SK하이닉스 | 뉴스룸 + 컨콜 | 상동 |
| **마이크론** | **SEC EDGAR 10-Q** | 상동 — **유일하게 문서화된 수치** |

마이크론 10-Q는 이 형식으로 적혀 있다:
> *"Sales of DRAM products increased X%, primarily due to a Y% range increase in average selling prices and a Z% range increase in bit shipments"*

→ `docs/05_비트그로스_ASP.md`의 표를 갱신하고, **ASP 기여율**을 다시 계산한다.
→ 계산식: 매출증가율 ≈ (1 + 비트성장률) × (1 + ASP상승률) − 1

### ② 밸류에이션 4종 — 같은 날 한꺼번에

**반드시 같은 날 재야 한다.** 시점이 다르면 비율(선행÷후행, 상대배수)이 계산되지 않는다. 코드로 막아뒀다.

| 필드 | 소스 |
|---|---|
| `soxx_pe_trailing` | iShares 공식 |
| `soxx_pe_forward` | 증권사 반도체 섹터 리포트 |
| `ndx_pe_trailing` | Siblis Research |
| `ndx_pe_forward` | Siblis Research |

### ③ 컨센서스·점유율

- 목표주가·추정 리비전 → 한경 컨센서스, stockanalysis
- HBM·DRAM 점유율 → Counterpoint 분기 트래커
  - ⚠ Counterpoint는 **과거 수치를 소급 수정**한다. 최신 표만 쓰고 과거 표와 섞지 않는다

### ④ 반증 신호 보드 재점검

`docs/04`, `docs/05`, `docs/06`의 반증 조건을 하나씩 대조한다.

## 7.5 문제가 생겼을 때

### Actions가 빨간 X

`Actions` 탭 → 실패한 실행 클릭 → 로그의 빨간 줄 확인.

| 메시지 | 원인 | 조치 |
|---|---|---|
| `[중단] manual JSON 문법 오류 — N행` | 쉼표·따옴표 실수 | 해당 줄 수정 |
| `news:... HTTP Error 404` | RSS 주소 변경 | `FEEDS` 목록 교체 |
| `fred — FRED_API_KEY 미설정` | Secret 누락 | Settings → Secrets 확인 |
| `price:... HTTP Error` | Stooq 일시 장애 | 대개 다음 날 자동 복구 |

**한 소스가 죽어도 나머지는 갱신된다.** 못 받은 값은 직전 값을 `직전값` 표시로 유지한다.

### JSON 문법 오류 — 가장 흔한 실패

증상 셋:
- 앞 항목 끝에 **쉼표 누락**
- 마지막 항목 뒤에 **쉼표 있음** (JSON은 금지)
- 따옴표 짝 안 맞음

**VS Code가 빨간 밑줄로 알려준다.** 저장 전에 파일을 훑어볼 것.

푸시 전에 직접 검사하려면:
```bash
python -c "import json;json.load(open('data/manual.json',encoding='utf-8'));print('OK')"
```

### git pull에서 CONFLICT

```bash
git rebase --abort
git pull --rebase
```

그래도 나면 충돌 파일이 `data/data.json`일 가능성이 높다. 이건 자동 생성 파일이라 버려도 된다:
```bash
git checkout --theirs data/data.json
git add data/data.json
git rebase --continue
```

## 7.6 절대 하지 않을 것

- **추정치를 레벨 필드에 넣지 않는다.** 모르면 비운다. 한 번 오염된 시계열은 되돌릴 수 없다
- **규격이 다른 가격을 이어붙이지 않는다.** DDR4 8Gb과 DDR5 16Gb는 다른 시계열이다
- **모듈 가격(USD/module)과 칩 가격(USD/chip)을 섞지 않는다**
- **소스를 바꾸지 않는다.** SOXX P/E는 iShares, 나스닥은 Siblis로 고정. 소스가 바뀌면 시계열이 끊긴다
- **과거 항목을 수정하지 않는다.** 틀린 값을 발견하면 지우지 말고 새 항목에 정정 기록을 남긴다
- **`dram_spot_history.json`을 건드리지 않는다.** 임포트 아카이브다

## 7.7 이 루틴이 지키는 것

주 5분의 목적은 정보 수집이 아니다.

> **손가락이 근질거릴 때, 근거로 쓸 숫자가 이미 쌓여 있게 하는 것.**

항목 01의 매도 원칙은 *"현물가 떨어지는데요"* 같은 소음에 반응하는 것을 경계한다. 그 소음과 신호를 구분하려면 **평소에 재놓은 시계열**이 있어야 한다. 급할 때 찾아보는 숫자는 이미 늦다.

기록이 쌓이면 두 가지가 생긴다.
1. 지금 값이 **비정상인지 아닌지** 판단할 기준선
2. 과거의 내 판단이 **맞았는지 틀렸는지** 검증할 근거

두 번째가 더 중요하다.
