# 3·4. 데이터 소스 맵 — 반도체 리얼타임 데이터 & 뉴스 경로

> 목적: 대시보드가 주기적으로 끌어올 **공신력 있는** 경로를 확정한다.
> 원칙 ① 1차 출처 우선(거래소·통계기관·기업 IR) ② 스크래핑보다 API/RSS ③ 유료는 대체재를 병기 ④ 갱신 주기를 명시.

**주의 — 신뢰도 등급**
- 🟢 1차: 거래소, 기업 IR, 통계기관(FRED/SIA/SEMI), ETF 발행사
- 🟡 2차: 증권사 리포트, TrendForce/Omdia, 로이터/블룸버그
- 🔴 배제: AI 자동생성 주가예측 사이트(`wonforecast` 류), 개인 블로그의 목표주가 재가공, 커뮤니티 Q&A. **컨센서스 수치를 이런 곳에서 인용하지 않는다.**

## 3.5 업데이트 주기 설계

| 주기 | 대상 | 액션 |
|---|---|---|
| **실시간~일** | 삼전·하닉·MU 주가, SOXX, 환율 | 자동 |
| **일** | DRAM 현물가, DXI | 자동/반자동 |
| **주 (월요일)** | 유진 Memory Watch, DXI 주간 등락, 스팟 프리미엄 | 리포트 확인 |
| **월** | WSTS 글로벌 매출, SEMI 빌링, FRED PPI, 한국 반도체 수출, DRAM 계약가 | 자동(FRED) + 수동 |
| **분기** | 3사 실적·ASP·Bit Growth·CapEx 가이던스, 컨센서스 리비전, SOXX PER | 수동 정독 |
| **수시** | 뉴스, 목표주가 변경 | RSS |

### 자동화 우선순위
1. **FRED API** — 유일하게 완전 무료·안정 API. 즉시 자동화 가능
2. **KRX OPEN API** — 국내 주가
3. **RSS 수집** — 뉴스 (Reuters, 전자신문, THELEC)
4. 나머지는 **링크 + 최종확인일 수동 갱신** 방식으로 대시보드에 표기 (라이선스 리스크 회피)

## 3.6 대시보드 구현 방침

현재 대시보드는 **정적 HTML**이므로 외부 API를 브라우저에서 직접 호출하면 CORS·API 키 노출 문제가 발생한다. 따라서:

- **v1 (현재)**: 소스별 **딥링크 + 수동 입력 스냅샷 + 최종 갱신일 표기**. 사용자가 클릭 한 번으로 원출처 확인.
- **v2 (권장)**: 서버(또는 Cloudflare Worker/GitHub Actions)에서 하루 1회 FRED·KRX·RSS를 수집해 `data.json`으로 커밋 → 대시보드가 그 파일만 fetch. API 키는 서버에 보관.

> ⚠️ 유료 데이터(TrendForce 계약가, Bloomberg 등)는 **자동 스크래핑 금지**. 링크와 요약만.
